21. Saida maka Baze-dadus : 1.1 Definisaun Baze-dadus, 1.2 Operasaun Báziku Baze-DadusEstudante sei komprende no iha konhesimentu kona-ba Baze-dadus: • Esplika definisaun no konseitu fundamental baze-dadus. • Deskreve operasaun báziku iha baze-dadus: Insert, Update, Delete no Select. • Distingue baze-dadus ho ficheiru tradisional. • Identifika vantajen uza baze-dadus iha sistema informasaun.
3(kontinuasaun) 1.3 Objetivu Baze-dadus 1.4 Implementasaun Baze-DadusEstudante sira sei komprende no iha konhesimentu kona-ba Objetivu no Implementasaun Baze-dadus: • Identifika objetivu prinsipál uza baze-dadus: redusaun redundansia, konsistensia dadus. • Komprende prosesu implementasaun baze-dadus iha ambiente real. • Analiza kazu implementasaun baze-dadus iha instituisaun edukasaun no negosiu. • Avalia dezafiu sira iha implementasaun baze-dadus.
42. Sistema Baze-dadus : 2.1 Komponente Sistema Baze-dadus 2.1.1 Hardware iha Sistema Baze-Dadus 2.1.2 Sistema Operasaun (Operating System) 2.1.3 Baze-dadus (Database)Estudante sira sei komprende no iha konhesimentu kona-ba Komponente Sistema Baze-dadus: • Identifika hardware ne'ebé presiza ba sistema baze-dadus. • Esplika papel sistema operasaun iha jestaun baze-dadus. • Komprende estrutura baze-dadus hanesan komponente sentral sistema. • Deskreve relasaun entre hardware, OS no baze-dadus.
5(kontinuasaun) 2.1.4 Sistema Jestaun Baze-Dadus (DBMS) 2.1.5 Utilizadór (User) 2.1.6 Aplikasaun seluk (Software)Estudante sira sei komprende no iha konhesimentu kona-ba DBMS no Utilizadór: • Esplika funsaun no papel DBMS iha jestaun baze-dadus. • Identifika tipu utilizadór: DBA, programador no utilizadór final. • Komprende papel software aplikasaun seluk iha sistema baze-dadus. • Analiza arkitetura kompletu sistema baze-dadus.
6(kontinuasaun) 2.2 Abstrasaun Dadus 2.3 Linguajen Baze-de-Dadus (Database Language) 2.4 Estrutura Sistema Báziku 2.3.1 Data Definition Language (DDL) 2.3.2 Data Manipulation Language (DML)Estudante sira sei komprende no iha konhesimentu kona-ba Abstrasaun no Linguajen Baze-dadus: • Esplika nivel abstrasaun dadus: nivel fiziku, konseitual no vizualizasaun. • Komprende DDL: CREATE, ALTER, DROP ba definisaun estrutura dadus. • Aplika DML: SELECT, INSERT, UPDATE, DELETE ba manipulasaun dadus. • Distingue DDL ho DML iha kontestu DBMS.•Komprende no iha konhesimentu kona-ba Estrutura Sistema Báziku: • Deskreve estrutura interna DBMS: jestaun buffer, armazenamentu no katalogo. • Komprende arkitetura kliente-servidor iha baze-dadus. • Identifika komponente prosesamentu query: parser, optimizer no executor. • Analiza prosesu tranzsasaun no kontrolu konsistensia.
73. Baze-dadus Relasionál : 3.1 Definisaun Baze-dadus Relasionál 3.2 Baze-dadus AkadémikuEstudante sira sei komprende no iha konhesimentu kona-ba Baze-dadus Relasionál: • Esplika konseitu modelu relasionál: relasaun, tupla no atributu. • Identifika propriedade tabela relasionál: chave primária, integridade. • Analiza baze-dadus akadémiku hanesan ezemplu prátiku: tabela estudante, kursu. • Komprende konseitu schema no instansia iha modelu relasionál.
8(kontinuasaun) 3.3 Operasaun no Lingua Baze-Dadus 3.4 Relasaun Entretabela 3.5 Baze-dadus ne'ebé di'akEstudante sira sei komprende no iha konhesimentu kona-ba Operasaun no Relasaun: • Aplika operasaun álgebra relasionál: selesan, projesaun, junasaun. • Komprende SQL hanesan lingua padraun baze-dadus relasionál. • Deskreve tipu relasaun entretabela: 1:1, 1:N, N:M. • Identifika karaterístika baze-dadus ne'ebé di'ak: konsistensia, integridade no seguransa.
94. Normalizasaun Dadus : 4.1 Atributu Tabela 4.1.1 Key Deskritivu no Atributu 4.1.2 Atributu Simples no Atributu KompostuEstudante sira sei komprende no iha konhesimentu kona-ba Atributu Tabela: • Distingue key (chave) ho atributu deskritivu iha tabela. • Identifika chave primária, chave kandidata no chave estrangeiru. • Esplika diferensa atributu simples ho atributu kompostu. • Aplika konsitu atributu iha dezain tabela baze-dadus.
10(kontinuasaun) 4.1.3 Atributu ho valór ida de'it no atributu ho valór barak 4.1.4 Atribut Harus Bernilai (Mandatory Attribute) no Nilai Null 4.1.5 Atributu DerivaduEstudante sira sei komprende no iha konhesimentu kona-ba Tipu Atributu: • Distingue atributu valor singular ho atributu multi-valor. • Komprende konseitu mandatory attribute no implikasaun NULL iha baze-dadus. • Esplika atributu derivadu no nia kalkulasaun baseia atributu seluk. • Aplika regra integridade entidade relasionadu ho NULL no mandatory.
11(kontinuasaun) 4.2 Tipu Domíniu no Dadus 4.3 Dependénsia FunsionálEstudante sira sei komprende no iha konhesimentu kona-ba Domíniu no Dependénsia Funsionál: • Identifika tipu dadus no domíniu: integer, varchar, date, boolean. • Esplika konseitu dependénsia funsionál (X→Y) no sira-nia importansia. • Identifika dependénsia funsionál kompletu, parsial no transitivu. • Analiza dependénsia funsionál iha tabela baze-dadus exemplu.
12(kontinuasaun) 4.4 Normalizasaun ho Dependénsia Funsionál 4.4.1 Dekomposisaun ho Lakon-Join 4.4.2 Prezervasaun DependénsiaEstudante sira sei komprende no iha konhesimentu kona-ba Normalizasaun: • Komprende objetivu normalizasaun: elimina redundansia no anomalia. • Aplika dekomposisaun lossless-join ba tabela baze-dadus. • Verifika prezervasaun dependénsia iha prosesu normalizasaun. • Analiza antes no depois normalizasaun iha tabela ezemplu.
13(kontinuasaun) 4.4.3 Boyce-Codd Normal Form (BCNF) 4.4.4 Forma Normál Etapa Datoluk (Forma Normál 3)Estudante sira sei komprende no iha konhesimentu kona-ba BCNF no 3NF: • Esplika kondisaun BCNF no diferensa ho 3NF. • Aplika algoritmu normalizasaun ba BCNF no 3NF. • Identifika kazu iha-ne'ebé BCNF la prezerva dependénsia. • Kompara BCNF no 3NF iha termu vantajen no limitasaun.
14(kontinuasaun) 4.5 Forma Normál seluk 4.5.1 Forma Normál 1 (1NF) 4.5.2 Forma Normál 2 (2NF) 4.5.3 Forma Normál 4 (4NF) no Forma Normál 5 (5NF)Estudante sira sei komprende no iha konhesimentu kona-ba Forma Normál sira: • Identifika violasaun 1NF no aplika normalizasaun ba 1NF. • Elimina dependénsia funsionál parsial hodi hetan 2NF. • Komprende 4NF no 5NF ba eliminasaun dependénsia multi-valor no join. • Aplika prosesu normalizasaun kompletu 1NF→2NF→3NF→BCNF→4NF.
15🔴EZAME MEDIU SEMESTRE (EMS)Teste Medio Semestre — Avalia materia semana 1 to 14.
165. Modelu Dadus : 5.1 Faktu Mundu Reál nian 5.2 Modelu Entidade-Relasaun 5.2.1 Entidade no Konjuntu Entidade 5.2.2 Atributu (Atributu / Propriedade)Estudante sira sei komprende no iha konhesimentu kona-ba Modelu E-R: • Esplika konseitu faktu mundu reál no nia representasaun iha baze-dadus. • Identifika entidade no konjuntu entidade iha senáriu real. • Deskreve atributu entidade: simples, kompostu, derivadu no multi-valor. • Distingue entidade ho relasaun iha modelu E-R.
17(kontinuasaun) 5.2.3 Relasaun no Konjuntu Relasaun 5.2.4 Kardinalidade/Grau Relasaun nianEstudante sira sei komprende no iha konhesimentu kona-ba Relasaun no Kardinalidade: • Esplika konseitu relasaun no konjuntu relasaun iha modelu E-R. • Identifika grau relasaun: unáriu, binairu no ternairu. • Komprende kardinalidade: 1:1, 1:N no M:N ho sira-nia ezemplu. • Aplika konsitu relasaun iha senáriu negosiu real.
18(kontinuasaun) 5.3 Diagram Entity-Relationship (Diagram E-R) 5.3.1 Etapa atu halo Diagrama E-R 5.3.2 Diagrama E-R ho Disionáriu DadusEstudante sira sei komprende no iha konhesimentu kona-ba Diagrama E-R: • Esplika simbol padraun iha diagrama E-R: retangulu, oval, losango. • Aplika etapa sistemátiku hodi kria diagrama E-R husi senáriu. • Kria disionáriu dadus (data dictionary) akompaña diagrama E-R. • Valida diagrama E-R kontra rekizitu sistema.
19(kontinuasaun) 5.3.3 Grau Mínreu Relasaun nian 5.3.4 Diagrama E-R iha Notasaun selukEstudante sira sei komprende no iha konhesimentu kona-ba Grau Mínreu no Notasaun: • Esplika grau mínreu (minimum cardinality): 0 no 1 ho sira-nia signifikadu. • Komprende notasaun Crow's Foot no UML iha diagrama E-R. • Kompara notasaun sira-nian no sira-nia uza iha industria. • Aplika notasaun Crow's Foot hodi kria diagrama E-R kompletu.
20(kontinuasaun) 5.4 Variante Entidade 5.4.1 Konjuntu Entidade Fraku 5.4.2 Sub-Entidade (Subtipu Entidade)Estudante sira sei komprende no iha konhesimentu kona-ba Variante Entidade: • Identifika entidade fraku no sira-nia dependénsia ba entidade forte. • Esplika partial key no nia papel iha identifikasaun entidade fraku. • Komprende sub-entidade (subtype) no super-entidade (supertype). • Aplika konseitu entidade fraku no sub-entidade iha diagrama E-R.
21(kontinuasaun) 5.5 Variante Relasaun 5.5.1 Relasaun Unáriu 5.5.2 Relasaun Multi-Entidade (Relasaun N-ary) 5.5.3 Relasaun Dupla (Redundant Relation)Estudante sira sei komprende no iha konhesimentu kona-ba Variante Relasaun: • Esplika relasaun unáriu (rekursiva) ho ezemplu: empregadu superviza empregadu. • Komprende relasaun N-ary ho tolu ka liu entidade. • Identifika relasaun redundante no sira-nia implikasaun. • Aplika variante relasaun sira iha diagrama E-R.
22(kontinuasaun) 5.6 Espesializasaun no Jeneralizasaun 5.7 AgregasiEstudante sira sei komprende no iha konhesimentu kona-ba Espesializasaun no Jeneralizasaun: • Esplika espesializasaun top-down no jeneralizasaun bottom-up. • Komprende kontradirsaun: disjoint vs. overlapping, total vs. parsial. • Deskreve agregasaun hanesan abstrasaun relasaun hanesan entidade. • Aplika espesializasaun, jeneralizasaun no agregasaun iha diagrama E-R.
23(kontinuasaun) 5.8 Prosesu Avansadu 5.8.1 Key Alternatif (Alternate Key) 5.8.2 Kodifikasaun Internu 5.8.3 Dekompozisaun no Normalizasaun Konjuntu Entidade nian 5.8.4 FleksibilitasEstudante sira sei komprende no iha konhesimentu kona-ba Prosesu Avansadu: • Identifika chave alternatifu no sira-nia uza iha dezain baze-dadus. • Esplika kodifikasaun internu hodi identifika rekord uniku. • Aplika dekomposisaun no normalizasaun ba konjuntu entidade. • Komprende konseitu fleksibilitas modelu dadus ba mudansa rekizitu.
24Revizaun no Prátiku Kria Diagrama E-R Kazu Estudu: Sistema Informasaun AkadémikuEstudante sira sei bele: • Aplika hotu-hotu konseitu modelu E-R iha kazu estudu real. • Kria diagrama E-R kompletu ba sistema informasaun akadémiku. • Identifika no resolve problema iha dezain modelu dadus. • Prezenta no defende eskolla dezain diagrama E-R.
256. Implementasaun Baze-dadus : 6.1 Transforma Modelu Dadus ba Baze-dadus Fíziku 6.1.1 Transformasaun Jerál/Báziku 6.1.2 Implementasaun ba Konjuntu Entidade Fraku no Sub-EntidadeEstudante sira sei komprende no iha konhesimentu kona-ba Transformasaun Modelu: • Aplika regra transformasaun jerál: entidade ba tabela, atributu ba koluna. • Implementa entidade fraku no sub-entidade iha tabela fíziku. • Komprende estratéjia implementasaun hierarkia: single table, joined, table-per-type. • Kria skripta SQL DDL hosi diagrama E-R.
26(kontinuasaun) 6.1.3 Implementasaun Relasaun Unáriu 6.1.4 Implementasaun Relasaun Multi-Entidade (Relasaun N-ary) 6.1.5 Implementasaun Relasaun RedundanteEstudante sira sei komprende no iha konhesimentu kona-ba Implementasaun Relasaun: • Implementa relasaun unáriu ho self-referential foreign key. • Transforma relasaun N-ary ba tabela assosiativu iha SQL. • Identifika no resolve relasaun redundante iha baze-dadus fíziku. • Kria no verifika foreign key constraints iha DBMS.
27(kontinuasaun) 6.1.6 Aplikasaun ba Espesializasaun no Jeneralizasaun 6.1.7 Implementasaun AgregasaunEstudante sira sei komprende no iha konhesimentu kona-ba Implementasaun Avansadu: • Aplika estratéjia implementasaun espesializasaun no jeneralizasaun. • Kompara estratéjia single-table, multi-table no hybrid ba hierarkia. • Implementa agregasaun hanesan relasaun entidade kompleksu. • Avalia impaktu eskolla implementasaun ba performansia query.
28(kontinuasaun) 6.2 DBMS no Estrutura Tabela 6.3 Índise sira no Estrutura Armazenamentu sira 6.3.1 Índise Primáriu 6.3.2 Índise Sekundáriu 6.3.3 Estrutura Armazenamentu Projeitu IntegraduEstudante sira sei komprende no iha konhesimentu kona-ba DBMS no Índise: • Komprende estrutura interna tabela iha DBMS (heap, B-tree, hash). • Esplika índise primáriu no nia papel iha aksesu dadus rápidu. • Identifika kazu uza índise sekundáriu ba query optimizasaun. • Kria no analiza índise uza SQL: CREATE INDEX. •Komprende no iha konhesimentu kona-ba Armazenamentu no Projeitu: • Deskreve estrutura armazenamentu: sekuensial, klustar no indexadu. • Komprende trade-off entre tipu estrutura armazenamentu. • Define problema baze-dadus real no rekizitu sistema. • Kria diagrama E-R kompletu ba projeitu integrado.
29(kontinuasaun) 6.3.3 Estrutura Armazenamentu Projeitu Integradu — Parte 1: Definisaun Problema, Koleta Rekizitu no Kria Diagrama E-REstudante sira sei komprende no iha konhesimentu kona-ba Armazenamentu no Projeitu: • Deskreve estrutura armazenamentu: sekuensial, klustar no indexadu. • Komprende trade-off entre tipu estrutura armazenamentu. • Define problema baze-dadus real no rekizitu sistema. • Kria diagrama E-R kompletu ba projeitu integrado.
